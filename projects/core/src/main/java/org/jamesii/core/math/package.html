Math package for the simulation framework.

General description

The math packages provide various mathematical functions more or less frequently required for modeling and simulation. They have been implemented and ship with the core to reduce the need of integrating external packages, and thus to reduce dependencies of projects based on the framework. Some of them are even required in the core, and the core has by convention, no dependencies to any external stuff.
However, if you prefer to use an external math package for your M&S application this is possible as well - then you can ignore this package.
Basically this package and its subpackages contain math stuff from as different domains as
  • parse trees
  • random numbers / random distributions
  • statistics
  • complex numbers
  • integration algorithms
  • interpolation
  • matrix handling
The random numbers for example are used throughout the framework and they are the only part which should not completely exchanged: a sophisticated handling of random number generation is mandatory for credible simulation results. If you are in need of different random number generators / different implementations of generators provided by the framework you should add these by using the corresponding plug-in type of the framework.
