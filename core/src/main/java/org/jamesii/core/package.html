Provides the core of the simulation framework.

The core of the framework mainly contains general extension points for common modelling and simulation jobs, e.g., random number generation, modelling paradigm, simulation algorithms. The management of extension points and of extensions is done by the {@link james.core.Registry} class. Whenever you need functionality you should query the registry to get an appropriate one. This can be done with one of the methods of the registry. The passed parameter block may contain further information used by the AbstractFactory of the plug-in type to find the best solution to the problem at hand. The Registry is not realized as singleton, but commonly it will be used as a singleton throughout the instance of the m&s framework. You can access the current version by the {@link james.SimSystem#getRegistry()} method.
